# 网页版五子棋游戏设计规格

**日期：** 2026-05-25  
**项目：** web_project/gomoku

---

## 1. 项目概述

### 1.1 目标
构建一个网页版五子棋游戏，支持人机对战，玩家可与不同难度的 AI 进行对局。

### 1.2 核心需求

| 项目 | 决策 |
|------|------|
| 对局模式 | 人机对战 |
| AI 难度 | 多级（简单 / 中等 / 困难） |
| 棋盘规格 | 多种可选（15×15、19×19、自定义） |
| 胜负规则 | 简单规则，先连五子者胜，无禁手 |
| 技术栈 | Vue 3 + Vite |
| 视觉风格 | 简约现代风 |
| 辅助功能 | 基础功能（棋盘、落子、胜负判断） |
| 先手选择 | 玩家自选执黑或执白 |
| 页面布局 | 左右布局 |

## 2. 整体架构

### 2.1 项目结构

```
gomoku/
├── index.html                  # 入口
├── package.json
├── vite.config.js
├── src/
│   ├── main.js                 # Vue 应用入口
│   ├── App.vue                 # 根组件（左右布局容器）
│   ├── components/
│   │   ├── GameBoard.vue       # 棋盘组件（Canvas 绘制）
│   │   ├── GameInfo.vue        # 右侧信息面板
│   │   └── StartDialog.vue     # 开局设置弹窗
│   ├── game/
│   │   ├── board.js            # 棋盘数据模型 + 落子/判胜逻辑
│   │   ├── ai.js               # AI 引擎（模式匹配 + 搜索）
│   │   └── patterns.js         # 棋型模式定义
│   └── styles/
│       └── main.css            # 全局样式
```

### 2.2 组件职责

| 组件 | 职责 |
|------|------|
| **App.vue** | 左右布局容器，管理游戏状态（未开始/对局中/已结束），协调子组件 |
| **GameBoard.vue** | 用 Canvas 绘制棋盘网格和棋子，处理点击落子事件，显示最后一手标记 |
| **GameInfo.vue** | 显示当前回合、双方已落子数、控制按钮（开始/认输） |
| **StartDialog.vue** | 开局设置弹窗：棋盘大小、AI 难度、执黑/白 |

### 2.3 数据流

```
玩家点击棋盘
  → GameBoard 发射落子事件
  → App 处理落子
    → board.js 更新棋盘数据
    → 判断胜负
    → AI 计算下一步
    → 更新棋盘
    → 判断胜负
  → GameBoard 重新渲染
```

## 3. 棋盘与游戏核心逻辑

### 3.1 棋盘数据模型（board.js）

使用一维数组表示棋盘：

```js
// 例如 15×15 棋盘
const board = new Array(15 * 15).fill(0)
// 值：0 = 空, 1 = 黑, 2 = 白
```

### 3.2 核心功能

| 函数 | 功能 |
|------|------|
| `placeStone(board, x, y, color)` | 在指定位置落子 |
| `isEmpty(board, x, y)` | 判断位置是否可用 |
| `checkWin(board, x, y)` | 从最后一手出发检查四方向是否五连 |
| `isBoardFull(board)` | 判断是否平局 |

### 3.3 胜负判断逻辑

只检查最后落子位置的四个方向，不扫描全盘：

```
最后一次落子在 (x, y)，颜色为 C：
  对每个方向（水平/垂直/两个对角线）：
    向正方向数连续同色棋子数
    向反方向数连续同色棋子数
    如果总数 >= 5，则获胜
```

### 3.4 游戏状态机

```
未开始
  ↓ [点击开始]
对局中
  ↓ [五连/平局]
已结束
  ↓ [点击开始]
对局中（重新开始）

对局中
  ↓ [点击认输]
已结束
```

### 3.5 棋盘绘制（GameBoard.vue + Canvas）

**Canvas 尺寸计算：**
```
总宽度 = 格子大小 × (规格 - 1) + 边距 × 2
总高度 = 格子大小 × (规格 - 1) + 边距 × 2
```

**绘制顺序：**
1. 白色背景
2. 灰色网格线
3. 星位标记点（仅 15×15 棋盘）
4. 黑白棋子
5. 最后一手高亮标记

**点击事件处理：**
- 将鼠标坐标转换为棋盘坐标
- 吸附到最近的交叉点
- 判断该位置是否为空

## 4. AI 引擎设计

### 4.1 棋型模式定义（patterns.js）

AI 通过识别棋型来评估局面，每个棋型有固定分值：

| 棋型 | 说明 | 分值 |
|------|------|------|
| 连五 | 五子连续 | 100000 |
| 活四 | 四子连，两端都空 | 10000 |
| 冲四 | 四子连，一端被堵 | 1000 |
| 活三 | 三子连，两端都空 | 1000 |
| 眠三 | 三子连，一端被堵 | 100 |
| 活二 | 二子连，两端都空 | 100 |
| 眠二 | 二子连，一端被堵 | 10 |

**识别方式：**

在四个方向（横/竖/左斜/右斜）上，提取以某个位置为中心、长度为 9 的线段，用正则匹配判断棋型：

```js
// 例如：提取 (x,y) 水平方向的 9 格线段
// "01110" → 活三
// "011110" → 活四
// "11111" → 连五
```

### 4.2 AI 评估函数（ai.js）

对每个空位打分，综合攻防：

```
评分(位置) = 攻击分 × 1.1 + 防守分

攻击分 = AI 在此落子后形成的棋型总分
防守分 = 阻止玩家在此落子后能形成的棋型总分
```

- 进攻优先（攻击分权重 1.1 倍）
- 遇到玩家冲四等威胁时必须防守（防守分优先级提升）

### 4.3 三个难度级别

| 难度 | 策略 | 搜索深度 | 说明 |
|------|------|---------|------|
| 简单 | 纯评分 + 随机扰动 | 0 层 | 只在候选点中选较优的，30% 概率选择次优解（模拟新手） |
| 中等 | Minimax + Alpha-Beta | 1-2 层 | 会预判玩家下一步反应 |
| 困难 | Minimax + Alpha-Beta + 棋型组合识别 | 2-3 层 | 能识别双活三等组合杀招 |

### 4.4 性能优化

| 优化策略 | 说明 |
|----------|------|
| 候选点筛选 | 只考虑已有棋子周围 2 格范围内的空位，大幅减少搜索量 |
| Alpha-Beta 剪枝 | 提前剪掉明显不优的分支 |
| 棋型缓存 | 避免重复计算相同局面的棋型 |

## 5. UI 交互与样式

### 5.1 页面布局（App.vue）

```
┌─────────────────────────────────────────────┐
│                   页面标题                    │
├──────────────────────┬──────────────────────┤
│                      │   当前回合：⚫ 黑棋    │
│                      │   ─────────────────  │
│      棋盘区域         │   黑棋已落：12 子      │
│    (Canvas)          │   白棋已落：11 子      │
│                      │   ─────────────────  │
│                      │   [ 开始新局 ]        │
│                      │   [   认输   ]        │
├──────────────────────┴──────────────────────┤
│                                             │
└─────────────────────────────────────────────┘
```

### 5.2 StartDialog 开局弹窗

游戏开始前弹出，包含三个选择项：

- **棋盘大小：** 15×15 / 19×19 / 自定义（输入框，范围 5~30）
- **AI 难度：** 简单 / 中等 / 困难
- **执子选择：** 我执黑（先手）/ 我执白（后手）

点击"开始"后弹窗关闭，游戏进入对局状态。

### 5.3 棋子样式（简约现代风）

| 元素 | 样式 |
|------|------|
| 棋盘背景 | 纯白 `#ffffff` |
| 网格线 | 浅灰 `#ddd`，1px |
| 黑棋 | 深灰 `#333`，无描边，实心圆 |
| 白棋 | 浅灰 `#e0e0e0`，无描边，实心圆 |
| 最后一手标记 | 棋子中心一个小圆点（黑棋白点，白棋黑点） |
| 悬停提示 | 鼠标移到空位显示半透明棋子预览 |
| 胜利连线 | 五连棋子之间画一条高亮线 |

### 5.4 交互流程

1. 进入页面 → 自动弹出开局设置弹窗
2. 选好参数点"开始" → 弹窗关闭，棋盘渲染，轮到先手
3. 玩家点击空位落子 → AI 自动响应 → 交替进行
4. 任一方五连 → 弹出胜利/失败提示 + "再来一局"按钮
5. 玩家随时可点"认输" → 弹出确认 → 对局结束
6. 对局结束后点"开始新局" → 重新弹出设置弹窗

### 5.5 响应式处理

- 小屏幕时左右布局自动切换为上下布局（棋盘在上，信息在下）
- Canvas 尺寸自适应容器宽度
